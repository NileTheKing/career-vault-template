# Career Vault — Claude Code Agent Guide

이 vault는 개발자 취업 지원 자료를 관리하는 Obsidian 노트 저장소다.
AI 에이전트는 이 파일을 먼저 읽고 구조와 규칙을 파악한 뒤 작업할 것.

---

## Vault 목적

JD가 주어지면 → 관련 무기(경험/수치)를 태그로 빠르게 필터링 → 자소서/포트폴리오 초안 작성

---

## 디렉토리 구조

```
career-vault/
├── projects/          ← 프로젝트별 원자화 무기 노트 (T1~Tn)
├── me/
│   ├── 경험/          ← 비기술 경험 노트 (E1~En)
│   └── 스킬/          ← 기술 스택별 요약
├── companies/         ← 기업별 JD 분석 + 지원서 (gitignored)
├── templates/         ← 노트 작성 템플릿
└── _meta/             ← 무기_마스터_목록.md, 방법론 문서
```

## 지원서 작성 4단계 프로세스

에이전트는 현재 어느 단계인지 파악 후 해당 단계에 맞는 작업만 수행할 것.

- **Step 1**: JD & 회사 심층 조사 → `companies/{회사명}/JD.md` 작성
- **Step 2**: 경험 쥐어짜기 — `_meta/무기_마스터_목록.md` 기반 키워드 매핑만 (텍스트 작성 금지)
- **Step 3**: 글 작성 — 자소서 문항별 텍스트 or 포트폴리오 섹션 작성
- **Step 4**: 피드백 루프 — 개발자 면접관 / HR / 서류심사자 입장 각각 시뮬레이션

세부 규칙: `_meta/PORTFOLIO_WRITING_RULES.md`, `_meta/JOB_APPLICATION_METHODOLOGY.md` 참조

---

## 포트폴리오/자소서 작성 핵심 원칙

- 개조식 bullet 필수 (서술형 금지)
- 역피라미드: 첫 줄에 수치/결과 배치
- 수치 없는 추상 표현 금지 ("크게 향상", "효율적으로" 등)
- 실제 구현 없는 내용 주장 금지

## 무기 노트 파일명 규칙

```
T{번호}_{기술명}_{주제}.md    예) T2_Kafka_트랜잭션_파이프라인.md
E{번호}_{경험명}.md           예) E1_기업협력_프로젝트.md
```

## Semantic Search (MCP)

Smart Connections MCP가 연결된 경우, grep 대신 벡터 검색 우선 사용:
- `search_notes("쿼리")` — 관련 무기 노트 검색
- `get_similar_notes("파일경로")` — 유사 노트 탐색
- `get_stats()` — 임베딩 현황 확인
